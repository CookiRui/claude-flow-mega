# claude-flow

> 让 Claude Code 拥有结构化项目认知 + 自主执行能力的开源框架。

## 这是什么？

两套互补的系统，让 Claude Code 从"每次对话从零开始"变成"像读完所有内部文档的新员工"：

1. **4 层项目配置** — Constitution → Rules → Skills → Commands，用渐进式披露控制 token 成本
2. **8 层自主执行引擎** — 目标审查 → DAG 分解 → 并行执行 → 三级验证 → 元学习

**特点**：
- 即拷即用的项目模板（替换 `{占位符}` 即可开始）
- 3 个实用工具脚本（持久化循环、代码地图、Lint 反馈闭环）
- 从 100+ 配置文件的实际项目中提炼的方法论

## 快速开始（5 分钟）

### 只用项目模板

```bash
# 1. 复制模板到你的项目
cp -r template/ your-project/

# 2. 替换所有 {占位符}
# 主要文件：CLAUDE.md, .claude/constitution.md, .claude/rules/coding-style.md

# 3. 定制 .claudeignore（已提供模板，按需调整）

# 4. 开始使用 Claude Code — 它会自动加载 constitution + rules
```

### 完整自主执行

```bash
# 1. 拷贝模板 + 脚本到你的项目
cp -r template/ your-project/
cp -r scripts/ your-project/scripts/

# 2. 配置 Hooks（自动 lint 反馈闭环）
# template/.claude/settings.json 已预配置，确保 scripts/lint-feedback.sh 在项目中

# 3. 运行持久化循环（跨会话自动恢复）
python scripts/persistent-solve.py "你的目标"
```

## 核心概念

### 4 层信息架构

```
┌──────────────────────────────────────────────────────┐
│  Constitution（宪法）                                  │
│  项目特有的核心约束，AI 不知道就会写错的规则              │
│  加载：每次对话 | 修改频率：极低 | 预算：< 150 行       │
├──────────────────────────────────────────────────────┤
│  Rules（规则）                                         │
│  宪法的补充细则，宪法推导不出来的具体约束                  │
│  加载：每次对话 | 修改频率：低 | 预算：< 200 行          │
├──────────────────────────────────────────────────────┤
│  Skills（技能）                                        │
│  具体框架/API 的使用模板和参考                           │
│  加载：按需触发 | 修改频率：中 | 预算：单个 < 100 行     │
├──────────────────────────────────────────────────────┤
│  Commands（命令）                                      │
│  用户可调用的多步骤工作流                                │
│  加载：用户主动调用 | 修改频率：低 | 预算：不限           │
└──────────────────────────────────────────────────────┘
```

**核心原则**：Context 即成本。每次对话的固定开销 = CLAUDE.md + Constitution + Rules，必须精打细算。

### 8 层执行引擎

```
┌───────────────────────────────────────────────────────────┐
│  目标审查层     可行性预判 / 歧义澄清 / tradeoff 告知     │
├───────────────────────────────────────────────────────────┤
│  目标层         模糊目标 → 可验证子任务 DAG + 分解预检     │
├───────────────────────────────────────────────────────────┤
│  元控制层       进度感知 / 分级升级 / 轨迹记录 / 元学习    │
├───────────────────────────────────────────────────────────┤
│  验证质量层     L1 自检 / L2 对抗验证 / L3 端到端          │
├───────────────────────────────────────────────────────────┤
│  资源预算层     token·时间·步数预算 + 双模型路由            │
├───────────────────────────────────────────────────────────┤
│  执行层         Try → Evaluate → Adjust + 冲突检测 + 残值  │
├───────────────────────────────────────────────────────────┤
│  安全网层       检查点 / 回滚 / 回归验证 / Lint 反馈闭环   │
├───────────────────────────────────────────────────────────┤
│  知识层         按需搜索 / 类比迁移 / 代码地图 / 双层记忆  │
└───────────────────────────────────────────────────────────┘
```

**80% 的任务不需要走完 8 层**——入口处先判断复杂度（S/M/L/XL），简单任务走快速路径。

### 复杂度路由

| 级别 | 判定条件                   | 走哪些层                         | 推荐模型 |
|------|----------------------------|----------------------------------|----------|
| S    | 单文件、无依赖、可自动验证 | 直接执行 + L1 验证               | Haiku    |
| M    | 2-5 文件、有明确验证条件   | 目标层 + 执行层 + 安全网 + L1/L2 | Sonnet   |
| L    | 跨模块、涉及架构决策       | 全 8 层                          | Opus     |
| XL   | 需要产品/战略层面判断      | 8 层 + 外部决策                  | Opus     |

## 模板内容一览

```
template/
├── CLAUDE.md                          # 根入口：架构速览 + 工作规范
├── .claudeignore                      # AI 忽略文件配置
└── .claude/
    ├── constitution.md                # 宪法模板（4-7 条核心约束 + 强制执行协议）
    ├── settings.json                  # 预配置 Hooks（lint 反馈闭环）
    ├── rules/
    │   └── coding-style.md            # 编码风格规则模板
    ├── skills/
    │   ├── _template/                 # Skill 模板（触发描述 + 组件速查 + 核心用法）
    │   │   ├── SKILL.md
    │   │   └── references/detail.md
    │   ├── tdd/SKILL.md               # 内置：TDD 测试驱动开发（强制执行）
    │   └── verification/SKILL.md      # 内置：完成前验证清单（强制执行）
    └── commands/
        ├── feature-plan-creator.md    # 需求分析 → 技术方案 → 微任务拆解
        └── bug-fix.md                 # Bug 诊断 → 回归测试 → 修复 → 经验固化
```

所有模板文件使用 `{placeholder}` 占位符，附带 `<!-- 填写指南 -->` HTML 注释说明。

## 工具脚本

### persistent-solve.py — 跨会话持久化循环

```bash
python scripts/persistent-solve.py "让游戏帧率稳定 60fps"
python scripts/persistent-solve.py "重构认证系统" --max-rounds 5 --max-time 1800
```

自动在 Claude Code 会话间循环：执行 → 保存 WIP → 恢复 → 继续，直到目标达成或触发熔断（最大轮次/时间/连续无进展）。

### repo-map.py — 代码地图生成器

```bash
python scripts/repo-map.py /path/to/project              # 输出 .repo-map.json
python scripts/repo-map.py /path/to/project --format md   # 输出 .repo-map.md
```

用正则提取类/函数/方法定义，统计引用关系并排序。在 L/XL 任务前生成，规划时查地图定位文件，减少 50-70% 搜索 token。支持 Python、TypeScript、JavaScript、C#、Java、Go、Rust。

### lint-feedback.sh — 双向 Lint/Test 反馈闭环

配置为 Claude Code Hook，实现：编辑 → 自动 lint → 失败 → 错误反馈给 AI → AI 自动修复 → 再 lint → 通过。自动检测项目类型（ESLint / Ruff / dotnet format / golangci-lint / Clippy）。

## 适用场景

- **中大型项目**：多模块、多人协作、有明确架构约束
- **长期维护项目**：需要 AI 持续理解项目演进
- **质量敏感项目**：需要 TDD、验证清单、回归测试保障
- **复杂任务**：跨模块重构、性能优化、新子系统开发

**不适用**：一次性脚本、快速原型、探索性实验（直接用原生 Claude Code 更高效）。
