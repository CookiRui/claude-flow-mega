## Unity Batch Mode CLI

```bash
# Compile check
bash .claude/scripts/unity-compile.sh

# Run EditMode unit tests
bash .claude/scripts/unity-editmode-test.sh
bash .claude/scripts/unity-editmode-test.sh --filter "{test-assembly-name}"

# Run PlayMode / game tests (--scene is required)
bash .claude/scripts/unity-game-test.sh smoke --scene {default-scene-path}

# Asset operations (scenes, prefabs, materials, folders)
bash .claude/scripts/unity-ops.sh <command> [json-file|-] [--overwrite]
```

Exit codes: 0=success, 1=operation failure, 2=timeout, 3=infrastructure error.

## .meta File Rules (CRITICAL)

- Every file and folder under `Assets/` MUST have a corresponding `.meta` file.
- `.meta` files are generated by Unity Editor only — NEVER create or edit them manually.
- Deleting a file requires deleting its `.meta` too; renaming requires renaming its `.meta` too.
- Missing `.meta` = broken asset references. Always verify after file operations.
- After creating new C# files, run `bash .claude/scripts/unity-compile.sh` — Unity generates `.meta` files during compilation.

## Component Patterns

### Inspector Fields
```csharp
// CORRECT — always use [SerializeField] private
[SerializeField] private float _moveSpeed = 5f;
[SerializeField] private Transform _targetTransform;

// WRONG — never expose public fields to Inspector
public float moveSpeed = 5f;
```

### Component Caching
```csharp
// CORRECT — cache in Awake/Start
private Rigidbody _rb;
private void Awake() { _rb = GetComponent<Rigidbody>(); }
private void FixedUpdate() { _rb.AddForce(Vector3.up); }

// WRONG — GetComponent in Update loop
private void Update() { GetComponent<Rigidbody>().AddForce(Vector3.up); }
```

### Lifecycle Order
- `Awake()` — self-initialization only (GetComponent, field setup)
- `Start()` — cross-component dependencies (FindObjectOfType, event subscriptions)
- `OnDestroy()` — cleanup (unsubscribe events, release resources)

## Performance Constraints

Hot paths (`Update`, `FixedUpdate`, `LateUpdate` and their call chains) must obey:

| Rule | Violation Example |
|------|-------------------|
| No heap allocations | `new List<T>()`, `new Vector3[n]` |
| No LINQ | `.Where()`, `.Select()`, `.ToList()` |
| No string operations | `string.Format`, `$""`, `+` concat |
| No GetComponent | `GetComponent<T>()` in Update |
| No boxing | Passing value types to `object` params |
| Use CompareTag | `gameObject.tag == "Player"` is wrong; use `CompareTag("Player")` |
| No foreach on non-generic IEnumerable | Causes implicit boxing allocation |

For frequently instantiated/destroyed objects, use object pooling.

## C# Naming Convention

| Element | Convention | Example |
|---------|-----------|---------|
| Namespace | `{project-namespace}.<Feature>` | `{project-namespace}.Gameplay` |
| Types & Methods | PascalCase | `PlayerController`, `Initialize()` |
| Local variables & parameters | camelCase | `moveSpeed`, `targetPosition` |
| Private fields | _camelCase | `_currentHealth`, `_isGrounded` |
| Constants | PascalCase | `MaxPlayerCount` |

MonoBehaviour filename MUST match class name.

## Assembly Definitions

- Each `Scripts/` subfolder should have its own `.asmdef` to minimize recompilation and enforce dependency boundaries.
- Respect dependency direction — lower-level assemblies must not reference higher-level ones.
