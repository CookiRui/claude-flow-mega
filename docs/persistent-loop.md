# 跨会话持久化循环 + WIP 机制

单次 Claude Code 会话有预算限制（token/时间/上下文窗口）。复杂任务做到一半预算耗尽怎么办？**不是停下来等人——而是自动恢复继续推进。**

---

## 核心问题：单会话的 3 个硬退出点

| 退出点            | 原因                    | 后果             |
|-------------------|-------------------------|------------------|
| `attempts >= MAX` | 单任务尝试 2-3 次后放弃 | 任务未完成就停止 |
| `budget` 耗尽     | token/时间/步数用完     | 整体目标未达成   |
| `escalate(L3)`    | 信心太低，交接给人类    | 等人来才能继续   |

更关键的架构限制：**Claude Code 是单会话模型**。会话结束后没有外部进程自动重启下一轮。

---

## 解决方案：两层循环架构

```
┌─────────────────────────────────────────────────────────┐
│  外层：持久化循环（跨会话）                               │
│  由外部调度器驱动，负责"不达目的不停止"                     │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │  内层：单会话执行（autonomous_solve）               │  │
│  │  在预算和上下文范围内尽力推进                        │  │
│  │  遇到硬约束 → 保存 WIP → 退出                      │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  外层检查：目标达成了吗？                                 │
│  ├─ 是 → 完成，退出循环                                  │
│  └─ 否 → 从 WIP 恢复，启动新会话，继续推进               │
└─────────────────────────────────────────────────────────┘
```

---

## 三种运行模式

| 模式       | 怎么用                                        | 适合场景             |
|------------|-----------------------------------------------|----------------------|
| **全自动** | `python scripts/persistent-solve.py "目标"`   | 已明确的 L/XL 级任务 |
| **半自动** | 每轮结束后人工恢复 WIP                        | 需要中途审查的任务   |
| **单次**   | 默认行为，预算耗尽即止                        | S/M 级任务           |

---

## 全自动模式：使用方式

```bash
# Run the persistent loop scheduler
python scripts/persistent-solve.py "Stabilize game frame rate at 60fps"

# The script will automatically:
#   Round 1: Launch Claude Code → execute until budget runs out → save WIP
#   Round 2: Resume from WIP → continue executing → save WIP
#   Round 3: Resume from WIP → goal achieved → exit
#
# Max 10 rounds, or stops when circuit breaker triggers

# Options
python scripts/persistent-solve.py "goal" --max-rounds 5
python scripts/persistent-solve.py "goal" --max-time 1800  # 30 min total
```

## 内层自救机制

当某个子任务多次失败时，内层会尝试 3 级自救：

```
失败 → 自救 1: 换一个完全不同的角度
  ↓ 仍失败
      → 自救 2: 把当前子任务拆得更细
  ↓ 仍失败
          → 自救 3: 深度搜索学习后重试
  ↓ 仍失败
              → 保存 WIP，标记 [NEED_HUMAN]
```

只有所有自救手段都用尽后，才会标记需要人类介入。

---

## 熔断保护

全自动模式有安全限制，防止无限烧钱：

| 熔断条件         | 阈值   | 触发后             |
|------------------|--------|--------------------|
| 最大轮次         | 10     | 停止循环           |
| 最大总时间       | 1 小时 | 停止循环           |
| 最大费用         | $5     | 停止循环           |
| 连续无进展       | 3 轮   | 停止，提示人工接手 |
| 同一错误重复出现 | 3 次   | 停止，提示换方向   |

## WIP 机制

WIP（Work In Progress）是持久化循环的状态存储格式。

### 保存时机

- 预算即将耗尽时
- 升级给人类时（Level 3）
- 用户主动要求保存
- 外层调度器触发的轮次结束时

### WIP 文件格式

```yaml
---
status: active              # active | paused | done
goal: "目标描述"
round: 3                    # 当前第几轮
auto_resume: true            # 是否允许自动恢复
---

## 已完成任务
- [x] 定位瓶颈（DrawCall 40%, GC 25%, Physics 15%）
- [x] 优化 DrawCall（800 → 320）

## 剩余 DAG
- [ ] 优化 GC（依赖: 无）
- [ ] 优化 Physics（依赖: 无）
- [ ] 最终验证（依赖: GC, Physics）

## 已尝试策略与残值
- DrawCall: SRP Batcher 失败（Built-in RP）→ 残值: 确认渲染管线类型 + DrawCall 统计脚本
- DrawCall: Static/Dynamic Batching 成功

## 活跃约束
- 不能切换渲染管线
- 不能降低画质

## 上轮退出原因
预算耗尽（token 用了 450k/500k）

## 下一步建议
1. 优先处理 GC（占比 25%，预期收益最大）
2. 搜索 GC 优化最佳实践
```

### WIP 恢复流程

1. 扫描 WIP 目录，找到 `status: active` 的文件
2. 读取 WIP 文件，恢复上下文
3. 从"下一步建议"开始执行，不重复已完成的工作
4. 更新 WIP 文件（轮次 +1，追加新完成的任务）

### 普通 WIP vs 持久化循环 WIP

| 字段               | 普通 WIP | 持久化循环 WIP |
|--------------------|----------|----------------|
| 任务标题和目标     | ✅       | ✅             |
| 已完成的工作       | ✅       | ✅             |
| 待解决的问题       | ✅       | ✅             |
| 相关文件路径       | ✅       | ✅             |
| round 轮次         | ❌       | ✅             |
| auto_resume 标记   | ❌       | ✅             |
| 剩余 DAG           | ❌       | ✅             |
| 已尝试策略与残值   | ❌       | ✅             |
| 活跃约束           | ❌       | ✅             |
| 退出原因           | ❌       | ✅             |
| 下一步建议         | ❌       | ✅             |
